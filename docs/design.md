## Problem Statement

천둥이 치면 궁금해진다: "우리 동네 사람들도 들었나? 어떤 반응이야?" 트위터에서 "천둥" 검색하면 반응이 나오지만, 지역 기반 + 실시간 + 전용 경험으로 엮어주는 제품은 존재하지 않는다. 번개 추적 앱(LightningMaps, My Lightning Tracker)은 데이터/경보 중심이지, 사람들의 반응은 없다. 이 빈 공간을 채운다.

## What Makes This Cool

1. **Thunder Wave 시각화**: 천둥 소리의 물리적 전파(~343m/s)를 반응 타임스탬프와 위치 데이터로 재현. 지도에서 파동이 퍼져나가며 반응 마커가 찍히는 것을 볼 수 있다. 어떤 앱에서도 본 적 없는 경험.
2. **공유된 순간의 연결감**: 정보가 아니라 감정. "나 혼자 들은 게 아니다"는 확인. 같은 동네 사람들과 3-5분 안에 만들어지는 즉석 커뮤니티.
3. **스크린샷 바이럴**: 지도 위 파동 + 이모지 마커 + 실시간 피드가 한 화면에. SNS에 공유하고 싶은 비주얼.

## Constraints

- 사이드 프로젝트 (1주일 안에 MVP 완성 목표)
- 웹앱 (PWA) — 링크 공유 용이, 앱스토어 불필요
- 익명 기반 (회원가입 없음)
- 한국 중심 (기상청 API 활용)

## Premises

1. 천둥/날씨 이벤트 발생 시 사람들이 실시간으로 반응을 올릴 것이다
2. 날씨 이벤트 순간에 앱을 열고 반응할 동기가 충분하다
3. 위치 기반 "같은 지역" 묶기만으로 의미 있는 커뮤니티 경험이 된다
4. PWA로 충분한 실시간 경험을 제공할 수 있다

## Cross-Model Perspective

Claude subagent의 독립적 분석:
- **가장 쿨한 미발견 버전**: "Thunder wave" — 반응 타임스탬프 + 위치로 소리 전파 파동을 시각화. 5km 떨어진 사람은 15초 뒤에 반응. 물리 현상이 소셜 데이터로 보인다.
- **핵심 흥분 포인트**: 지도는 기능이 아니라 감정적 보상 — "나 혼자 들은 게 아니다"는 증거
- **50% 해결 오픈소스**: Mapbox GL JS + Supabase Realtime. 나머지: geo-bucketing (H3), 반응 UX, 스팸 방지, wave 애니메이션
- **주말 빌드**: Next.js PWA + Supabase 1테이블 + Mapbox. 이모지 5개 버튼. 탭→insert→실시간 구독→마커 애니메이션. Vercel 배포.

## Approaches Considered

### Approach A: Thunder Tap — 극단적 미니멀
- 이모지 5개 + 지도 1개. Supabase + Mapbox. 1일.
- Pros: 가장 빠른 검증, 배포 5분
- Cons: 히스토리 없음, 날씨 감지 없음

### Approach B: Thunder Community — 소셜 피드 + 지도
- 실시간 텍스트/이모지 피드 + 지도 + 기상청 API. 2일.
- Pros: 풍부한 경험, 날씨 이벤트 자동 감지
- Cons: API 연동 복잡도, 텍스트 모더레이션

### Approach C: Thunder Wave — 시각적 차별화 (CHOSEN)
- Approach B + thunder wave 애니메이션. 1주일.
- Pros: 스크린샷 찍고 싶은 경험, 바이럴 가능성 최고, 유일무이
- Cons: 애니메이션 개발 시간, 반응 수 적으면 wave 안 보임

## Recommended Approach

**Approach C: Thunder Wave** — 처음부터 시각적으로 차별화된 경험을 만든다. 사이드 프로젝트이므로 "검증"보다 "만들고 싶은 것"을 만드는 게 맞다.

### Tech Stack
- **Frontend**: Next.js 14+ (App Router) + PWA
- **Backend/DB**: Supabase (Postgres + Realtime subscriptions)
- **Map**: Mapbox GL JS
- **Deploy**: Vercel
- **Weather API**: 기상청 Open API (낙뢰 데이터)

### Core Architecture

```
[Browser] ←WebSocket→ [Supabase Realtime]
    ↓                        ↓
[Mapbox GL]           [Postgres]
    ↓                        ↑
[Wave Animation]      [기상청 API] (cron)
```

### Key Components

1. **Reaction System**
   - 테이블: `reactions(id, lat, lng, emoji, text, created_at, h3_index)`
   - 이모지 원탭: ⚡😱🙉😂🌧️
   - 텍스트 입력 (선택)
   - 익명 (localStorage UUID, 최초 방문 시 생성. 서버측 rate limit: UUID당 30초 1회, Supabase Edge Function에서 enforce)

2. **Geo-Bucketing**
   - H3 hexagonal grid (resolution 5, ~252km² cells, 반경 ~9km) — "같은 동네" 느낌 + 충분한 반응 수 확보
   - Supabase Realtime Postgres Changes 채널, `filter: "h3_index=eq.{userH3}"` 으로 구독
   - 사용자 위치 기반 자동 지역 설정 (인접 hex도 구독하여 경계 이슈 해결)

3. **Thunder Wave Animation**
   - **중심점 결정**: 기상청 낙뢰위치정보 API에서 개별 낙뢰 좌표 사용. **Fallback**: API가 좌표를 못 주면, 해당 시간대 첫 5개 반응의 위치 centroid를 wave 중심으로 사용 (사용자 반응 기반 추정)
   - 동심원이 343m/s 속도로 확장 (실제 음속 매핑)
   - **Pop-in 트리거**: 반응 마커는 `created_at` 타임스탬프를 기준으로, wave 링이 해당 거리에 도달하는 시뮬레이션 시간과 일치할 때 pop-in. 즉, wave 링이 지나간 위치의 마커만 보이고, 아직 안 도달한 위치는 숨김
   - **5분 폴링 갭 대응**: wave 애니메이션은 "리플레이" 모드로도 동작. 사용자가 앱을 열면 최근 5분 이내 이벤트를 가속 리플레이(10배속)로 보여줌
   - Mapbox GL JS custom layer + requestAnimationFrame

4. **Real-time Feed**
   - Supabase Realtime subscription (h3_index 필터)
   - 새 반응 slide-in 애니메이션
   - 거리 표시 (사용자 위치 기준)

5. **Weather Event Detection**
   - 기상청 낙뢰위치정보 서비스 (`getVilageFcst` 또는 `getLgt`) — Day 1에 API 키 발급 및 응답 확인 필수
   - Supabase Edge Function에서 3분 주기 cron 폴링
   - 낙뢰 감지 시 `weather_events` 테이블에 기록 → Realtime으로 클라이언트에 알림
   - **에러 핸들링**: API 실패 시 사용자 반응 급증(30초 내 10+개)을 낙뢰 이벤트 proxy로 활용

   > **MVP scope: 천둥/번개만.** 폭우, 눈, 황사, 태풍은 Future Directions 참조.

## Open Questions

1. **콜드 스타트**: 천둥이 안 칠 때 앱에 들어오면? → 최근 이벤트 히스토리 보여주기? 시뮬레이션 모드?
2. **스팸/어뷰즈**: 서버측 rate limit (UUID당 30초 1회, Edge Function enforce). IP 기반 fallback.
3. **기상청 낙뢰 API 좌표 정밀도**: Day 1에 반드시 확인. 개별 좌표 불가 시 사용자 반응 centroid를 wave 중심으로 사용 (Fallback 설계 완료).
4. **기상청 API 무료 호출 한도**: 일 10,000회 (3분 주기 = 일 480회, 충분)
5. **모바일 UX**: PWA에서 위치 권한 요청 타이밍과 UX

## Success Criteria

- [ ] 천둥이 칠 때 지도에서 wave 애니메이션이 보인다
- [ ] 이모지 탭 → 1초 내 지도에 마커 표시 + 피드에 반영
- [ ] 같은 지역(H3 res5, ~9km 반경) 사용자만 서로의 반응을 본다
- [ ] SNS 공유 링크가 동작한다 (OG 이미지 포함)
- [ ] 모바일에서 사용 가능한 반응형 레이아웃

- "천둥 들었는데 주변 사람들은 어떤 반응을 하는지 궁금함" — 기능 스펙이 아니라 감정에서 출발했다. 가장 좋은 제품은 다 이렇게 시작한다.
- 10x 버전에서 소리 자동감지는 빼고 다양한 날씨 이벤트 + 히스토리를 골랐다. 기술적으로 복잡한 것보다 경험의 폭을 넓히는 걸 선택한 것. 좋은 판단.
- Approach 선택에서 미니멀(A) 대신 바로 Thunder Wave(C)를 골랐다. 사이드 프로젝트에서 "검증"보다 "만들고 싶은 것"을 우선한 건 맞는 태도다. 재미가 없으면 사이드 프로젝트는 완성되지 않는다.
